## Language & Runtime

**Go.** Already in the project (fly-proxy), compiles to a single zero-dependency binary, cross-compiles trivially (`GOOS=darwin GOARCH=arm64 go build`).

## Distribution

- **goreleaser** — one config, builds binaries for every OS/arch, attaches to GitHub Releases
- **Homebrew tap** — `brew install kurtpayne/tap/yoke` (goreleaser generates the formula)
- **`go install`** — `go install github.com/kurtpayne/yoke/cli@latest`
- Skip npm/Docker — overkill for an API wrapper

## Framework

- **Cobra** — subcommands, shell completions (zsh/bash/fish), man page generation
- **Lipgloss** — color/styling without a full TUI framework
- **Viper** (Cobra built-in) — config file support

## Core Commands

```
yoke analyze stripe.com        # full analysis, pretty-printed
yoke analyze stripe.com --json # raw JSON output
yoke compare github.com gitlab.com  # side-by-side scores
yoke watch stripe.com          # re-analyze on interval, diff changes
yoke score stripe.com          # just the composite score + grade
```

## Features

### SSE Streaming in Terminal
Pipe the same EventStream the browser uses. Render check results as they arrive with colored status indicators. Feels live instead of "wait 8 seconds then dump."

### ASCII Radar Chart
The 5-axis score as a terminal-rendered radar. ~50 lines of code. Great screenshot for launch/README.

### Smart Output Detection
Detect TTY: if stdout is a terminal, pretty-print with colors. If piped, emit raw JSON. `yoke stripe.com | jq .ssl` just works, `yoke stripe.com` looks nice.

### Compare Mode
Side-by-side scores in a table, diff-colored (green = better, red = worse).

### Watch Mode
`yoke watch stripe.com` — re-analyze on interval, diff the output, alert on score changes. Seed of the monitoring/paid tier.

### `.yokerc` Config
YAML config for BYO API key, default format (json/table/compact), color theme. Cobra + Viper handles this natively.

### Shell Completions
`yoke completion zsh/bash/fish` — generated by Cobra for free.

## Scope

~400-500 lines of Go. The JSON API does all the heavy lifting — the CLI is a pretty face on `curl yoke.lol/domain.com`.

## Location

`yoke-public/cli/` in the monorepo, alongside `worker/`, `client/`, `fly-proxy/`, `extension/`.
